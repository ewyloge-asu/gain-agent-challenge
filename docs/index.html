What this is

A packaged folder of instructions and scripts you install into an AI agent.

An Agent Skill is a folder of instructions plus scripts that you install into an AI agent such as Claude. Once it is installed, you ask your questions in plain English. The agent runs the analysis for you and returns the answers, with a source for every number. You do not write any code.

The corpus

The records being investigated: 100K+ Senate lobbying filings, 108K House filings, and 141K congressional press releases. The files are inconsistent and too large to read by hand.

The skill

Plain Python scripts that clean up the records, match the same person or firm across the datasets, and answer questions about them. They give the same answer every time you run them.

The receipt

Every row carries a prov id. Any number the agent reports can be traced back to the exact filing or press release it came from, so a person can check it.

The core question

Pay → Say → Act

The skill lines up three things about a lawmaker that normally live in three separate piles of paperwork — the money that flows to them, what they tell the public, and what they legislate — so a reporter can see where they line up and where they don't.

PAY

Who funds them

Lobbyist and registrant contributions (LD-203), resolved to the member who received them, ranked into a "gatekeeper" leaderboard.

xref.py gatekeeper --filer lobbyist --year 2025
SAY

What they tell the public

141K press releases tagged into policy domains, turning a release count into a topic profile — then compared against what they actually sponsor.

xref.py say-vs-do "Mark Warner"
ACT

What they legislate

Sponsored-bill policy areas (Congress.gov) and dated contributions interleaved on a timeline — the "pay → say → act" cadence for one member.

xref.py timeline "Mark Warner" --year 2025

Under the hood

A four-stage pipeline, standard library only.

There is nothing extra to install. The skill is plain Python, and it runs in four steps: it loads the records, matches up the same people and firms, answers your questions, and lets you check every answer against the original source.

01 · ingest

Normalize

Load the House, Senate, and press files into one database, and label each row with where it came from.

02 · resolve

Match entities

Figure out when a firm, person, or lawmaker in one file is the same one in another file.

03 · xref

Cross-reference

Run set commands that answer questions and return totals plus a pointer to each source record.

04 · review

Verify

Take any answer's source id and open the exact filing or press release behind it before publishing.

New to AI agents

Your first investigation, step by step.

No jargon. This is what it feels like to use the skill the first time.

  1. Pick the question, in plain wordsTell the agent what you're chasing the way you'd tell a colleague: "Who's putting the most lobbyist money behind the lawmaker who runs the health committee?"
  2. Let it do the readingThe agent runs the skill's commands across hundreds of thousands of filings — the boring part — and comes back with a tidy ranking, profile, or timeline instead of a wall of files.
  3. Read the "say vs. do" gapAsk it to line up what a member talks about against what they sponsor. Where the two diverge is a lead to chase — not a conclusion, a starting point.
  4. Check the receipts before you trust itEvery figure comes with a provenance id. Hand one back to the agent and it shows you the exact filing or press release. If it doesn't say what the agent claimed, you caught it before print.
  5. Follow the threadThe skill keeps investigation state across sessions, so leads you flagged today are waiting when you come back tomorrow.
Get it · runs anywhere, not just here

Install the skill into your agent.

It's a portable Agent Skill — a self-contained folder you drop into the skills directory of any agent that supports the spec. Once installed, the agent discovers it on its own and you just ask in plain language. It ships with a ~6 MB demo corpus, so it runs out of the box before you point it at the full dataset.

# 1 · install the folder into your agent's skills directory
unzip lobbying-influence-mapper-1.1.0.zip -d ~/.agent/skills/

# 2 · try it immediately on the bundled demo (no key, no full corpus)
cd ~/.agent/skills/lobbying-influence-mapper
export GAIN_DATA_DIR="$PWD/demo_data" GAIN_WORKDIR="$PWD/build"
python3 scripts/ingest.py --years 2025 2026 --datasets senate contributions press
python3 scripts/resolve_entities.py

# 3 · or just ask your agent, in plain words —
#   "Who got the most lobbyist money in 2025, and does the top
#    senator's messaging match what they sponsor? Show me the source."
# For real work, point GAIN_DATA_DIR at the full federal corpus (same layout).

Comfortable with tooling

The command surface.

Standard-library Python, no install step. These run against the bundled demo or the full corpus — same commands, just change GAIN_DATA_DIR.

Build & query

# build the store + resolve entities (or: run_all.sh)
python ingest.py --years 2022 2023 2024 2025 2026 \
  --datasets senate contributions press
python resolve_entities.py

# PAY — who funds the gatekeepers
python xref.py gatekeeper --filer lobbyist --year 2025

# SAY — press topic profile + say-vs-do
python xref.py say "Mark Warner"
python xref.py say-vs-do "Mark Warner"

# ACT — money + press + bills on one timeline
python xref.py timeline "Mark Warner" --year 2025

# integrity + footprint
python xref.py anomaly --year 2025 --factor 5
python xref.py client "NIPPON STEEL"

Enrich & verify

# external data, tiered (keys optional)
python connector.py annotate-gatekeeper --year 2025  # no key
python connector.py fara                              # no key
python connector.py enrich-votes --year 2025          # Voteview
python connector.py enrich-act --year 2025            # Congress.gov
python connector.py enrich-finance --year 2025        # FEC

# coordinated messaging across offices
python detect_coordination.py clusters \
  --start 2025-01-01 --end 2025-03-31 --min-members 6

# VERIFY — provenance id back to the source record
python review.py \
  "data/congress_press/2026-01.jsonl#L2701"
Reproducibility ladder

Keys never gate the core findings.

The core findings run with no API keys and no internet, because the outside data they need is saved as snapshot files inside the package. API keys are optional and only unlock the extra tiers below. The full raw dataset is not included, so either use the bundled demo data or point GAIN_DATA_DIR at the dataset.

Guardrails

Built-in limits that keep findings honest and defensible.

Disclosure ≠ truth

Filings are self-reported, and about 35% report no income. A finding describes what was disclosed, not necessarily what happened.

Press is Senate-skewed

There are 1,000+ releases for active senators but only a handful for many House members. The "say" leg is reliable for senators and thin for low-volume members.

Topic tags are coarse

Press topics are tagged by keywords in the title and lede. A say-vs-do gap is a lead to confirm, not a verdict of hypocrisy.

Correlation, not cause

Money rising as a member takes a gavel is suggestive, not proof of cause. The skill shows the timeline and leaves the judgment to the reader.
