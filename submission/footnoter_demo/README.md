# Footnoter end-to-end demo (`howard-center-footnoter`)

The claim-hardening leg of the assembly line, run for real on this submission's own
findings: a `.docx` excerpt of the findings report, footnoted to the **actual pipeline
outputs** that back each number — as Word tracked changes plus margin comments.

## What's here

| File | What it is |
|---|---|
| `findings_draft.docx` | 4-paragraph draft excerpt of `../findings_report.md` (15 sentences, 11 flagged as claims) |
| `sources/` | 6 source records generated by the analysis engine this session (gatekeeper leaderboard, anomaly scan, Guthrie committee roles, Loc Nation raw filing via `review.py`, Tencent footprint, FEC denominator) |
| `claims.json` | output of `extract_claims.py` (deterministic claim extraction) |
| `sources.json` | output of `ingest_sources.py` (source index) |
| `matches.json` | the **agent-judgment step**: each claim matched to a verbatim supporting quote + surrounding context from a source |
| `flags.json` | the two concerns that need a human: a `no_source` (DoD China-list date is real-world context, not a corpus fact) and a `context_concern` (the ~5× Tencent multiplier isn't in the 2025-only extract) |
| `footnotes.json` | output of `format_footnote.py` (Howard-Center-format footnote text) |
| `findings_draft_footnoted.docx` | **the deliverable** — 10 footnotes inserted as tracked changes (9 sourced + 1 ⟨NEEDS SOURCE⟩ placeholder) and 2 margin comments |

## Result summary (from `insert_tracked_footnotes.py`)

```
inserted: 10 · needs_source: 1 · comments_attached: 2
```

Open `findings_draft_footnoted.docx` in Word with Track Changes visible: every claim's
footnote shows the verbatim engine output that supports it, the unsupported real-world
claim carries a `[No source found]` margin comment, and the partially-supported claim
carries a `[Possible out-of-context quoting]` comment.

## Reproduce

```bash
SK=../../menu/skills/howard-center-footnoter/scripts     # needs python-docx + lxml
python3 $SK/extract_claims.py findings_draft.docx --out claims.json
python3 $SK/ingest_sources.py sources --out sources.json
# (agent judgment: write matches.json + flags.json from claims × sources)
python3 $SK/format_footnote.py --matches matches.json --sources sources.json --out footnotes.json
python3 $SK/insert_tracked_footnotes.py --draft findings_draft.docx --claims claims.json \
    --footnotes footnotes.json --flags flags.json --out findings_draft_footnoted.docx \
    --author "investigative-desk (footnoter)"
```

Note: `claim_0012` is a sentence-splitter artifact ("Rev. Dr. Christina L. / Clement…"
split at the initial); the full sentence is footnoted once via `claim_0011`.
